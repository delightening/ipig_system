## 1. AUP 模組 GLP 合規分析

### 1.1 已符合項目

| 法規條款 | 系統功能 | 位置 |
|----------|---------|------|
| **GLP §7 試驗計畫內容** | 計畫書含 11 個區段：基本資訊、目的、試驗項目、設計、手術、人員、動物、指引、附件、簽章 | `types/protocol.ts:44-285` |
| **GLP §7 GLP 標記** | `is_glp: boolean` 旗標 + 登記機關選擇器 (FDA/CE/TFDA/CFDA) | `protocol.ts:46, 33-41` |
| **GLP §7 Study Director 欄位** | `basic.sd: { name, email }` 欄位存在 | `protocol.ts:76-79` |
| **GLP §7 研究編號** | `apply_study_number` 由 IACUC_STAFF 編輯 | `protocol.ts:87-96` |
| **GLP §5 3Rs — Replacement** | 替代方案搜尋平台 (altbib, db_alm, re_place) + 關鍵字 + 結論 | `protocol.ts:88-95` |
| **GLP §5 3Rs — Reduction** | 統計設計、樣本量方法、分組計畫 | `protocol.ts:96-107` |
| **GLP §5 3Rs — Refinement** | 疼痛分級 + 管理計畫 + 人道終點 + 限制措施 | `protocol.ts:178-193` |
| **GLP §7.2 修正案** | Major/Minor 分類 + 完整工作流 (Draft→Submitted→Classified→Approved) | `models/amendment.rs:11-33` |
| **GLP §8 版本控制** | `ProtocolVersion` 不可變快照 + 26 種活動類型追蹤 + Optimistic locking | `models/protocol.rs:93-227` |
| **GLP §8 電子簽章** | 密碼 + 手寫雙模式簽署，含身份認證與時間戳 | `handlers/signature/protocol_review.rs:35-86` |
| **GLP §2 多階段審查** | PreReview → VetReview → UnderReview → Approved，含 2-3 位指定審查委員 | `services/protocol/review.rs:51-88` |
| **GLP §8 審查留言** | 巢狀留言系統含草稿回覆 + PI 回應機制 | `models/protocol.rs:288-315` |

### 1.2 關鍵缺口

| # | 缺口 | GLP 條款 | 嚴重度 | 說明 |
|---|------|---------|--------|------|
| AUP-1 | **Study Director 為文字欄位** | §1.2 | P0 | SD 僅為 `{name, email}` 文字，非 user_id 參考；無法驗證角色權限或強制簽署 |
| AUP-2 | **核准無 QAU 簽核閘門** | §2 | P0 | 計畫書可直接從 UnderReview → Approved，無 QAU 確認步驟 |
| AUP-3 | **核准後無鎖定機制** | §8 | P1 | 簽署後文件仍可編輯，無 "locked/finalized" 不可變狀態 |
| AUP-4 | **修正案無影響評估欄位** | §7.2 | P1 | Amendment 模型缺少 `impact_assessment` 結構化欄位 |
| AUP-5 | **無從計畫書自動產生最終報告** | §9 | P1 | StudyFinalReport 為獨立實體，未連結計畫書內容自動帶入 |
| AUP-6 | **重大修正無 QAU 審查** | §2, §7.2 | P1 | Major amendment 核准前無 QAU 確認閘門 |
| AUP-7 | **無保存期限強制** | §10 | P2 | 已關閉計畫書無保存年限設定與刪除保護 (GLP 要求 5 年) |
| AUP-8 | **獸醫審查表無強制完成驗證** | §5 | P2 | VetReviewForm 各項目無強制全部填答的驗證邏輯 |

## 2. HR 模組台灣勞基法合規分析

### 2.1 已符合項目

| 法規條款 | 系統功能 | 位置 |
|----------|---------|------|
| **勞基法 §43 婚假** | `MARRIAGE` 假別已定義 | `models/hr.rs:184-195` |
| **勞基法 §43 喪假** | `BEREAVEMENT` 假別已定義 | 同上 |
| **勞基法 §43 產假** | `MATERNITY` 假別已定義 | 同上 |
| **性平法 陪產假** | `PATERNITY` 假別已定義 | 同上 |
| **性平法 生理假** | `MENSTRUAL` 假別已定義 | 同上 |
| **勞基法 §38 特休假** | `ANNUAL` 假別 + 到期追蹤 + 2年效期 | `services/hr/balance.rs:159-179` |
| **補休假管理** | 自動產生補休時數 + 1年到期追蹤 + 過期時數紀錄 | `migrations/008_hr_system.sql:110-128` |
| **多階核准流程** | L1→L2→HR→GM 四級核准鏈 + 審核紀錄 | `services/hr/leave.rs:317-321` |
| **出勤追蹤** | 打卡含 GPS + IP + 裝置 + 修正稽核 | `models/hr.rs:15-41` |
| **0.5 小時計算精度** | 時數計算四捨五入至 0.5 小時 | `services/hr/overtime.rs:40-44` |

### 2.2 關鍵缺口

| # | 缺口 | 法規條款 | 嚴重度 | 說明 |
|---|------|---------|--------|------|
| HR-1 | **加班費乘數錯誤** | 勞基法 §24 | **P0** | Type C (國定假日) 乘數為 1.66x，**法規要求 2.67x**；Type A (平日) 無分段計算 (前 2hr 1.34x、後 1.67x) |
| HR-2 | **特休假未自動計算** | 勞基法 §38 | P1 | `seniority_years` 欄位存在但未用於自動核算；需手動建立年度特休額度 |
| HR-3 | **無每月加班上限驗證** | 勞基法 §32 | P1 | 無 46 小時/月上限驗證邏輯（特殊情況 54 小時） |
| HR-4 | **無每日/每週工時驗證** | 勞基法 §30 | P1 | 無 8 小時/日、40 小時/週標準工時驗證與超時警告 |
| HR-5 | **無例假/休息日追蹤** | 勞基法 §36 | P2 | 無固定休息日 + 例假日的排班與追蹤機制 |
| HR-6 | **加班 Type A 無分段** | 勞基法 §24 | P1 | 平日加班應分「前 2 小時 1.34x」與「超過 2 小時 1.67x」兩段計算 |

### 2.3 加班乘數對照表

| 類型 | 現行系統 | 法規要求 | 差異 |
|------|---------|---------|------|
| A — 平日加班 (≤2hr) | 1.0x | **1.34x** | **嚴重低估** |
| A — 平日加班 (>2hr) | 1.0x | **1.67x** | **嚴重低估** |
| B — 休息日加班 | 1.33x | 1.33x | 正確 |
| C — 國定假日加班 | 1.66x | **2.67x** | **嚴重低估** |
| D — 天災加班 | 2.0x | 2.0x | 正確 |

> **嚴重警告**：HR-1 涉及薪資計算錯誤，可能導致法律糾紛與勞動檢查處罰。

### 2.4 勞基法 §38 特休假自動計算規則

| 年資 | 法定特休天數 |
|------|------------|
| 滿 6 個月未滿 1 年 | 3 天 |
| 滿 1 年未滿 2 年 | 7 天 |
| 滿 2 年未滿 3 年 | 10 天 |
| 滿 3 年未滿 5 年 | 14 天 |
| 滿 5 年未滿 10 年 | 15 天 |
| 滿 10 年以上 | 每年加 1 天，上限 30 天 |

系統目前需手動輸入 `entitled_days`，應改為依 `hire_date` 自動計算。
