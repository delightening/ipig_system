## 1. Backend (`backend/src/`)

### 1.1 應用程式核心

```
backend/src/
├── main.rs                          # 應用程式入口（精簡版，148 行）
├── lib.rs                           # Library crate 入口
├── config.rs                        # 組態設定（環境變數讀取等）
├── constants.rs                     # 常數定義
├── error.rs                         # 統一錯誤型別 (AppError)
├── routes/                          # 路由定義（依業務域拆分）
│   ├── mod.rs                       # 組裝所有子 Router
│   ├── admin.rs                     # 管理後台路由
│   ├── animal.rs                    # 動物管理路由
│   ├── auth.rs                      # 身份驗證路由
│   ├── erp.rs                       # ERP 進銷存路由
│   ├── hr.rs                        # 人資路由
│   ├── notification.rs              # 通知路由
│   ├── protocol.rs                  # 實驗計畫路由
│   ├── report.rs                    # 報表路由
│   ├── upload.rs                    # 檔案上傳路由
│   └── user.rs                      # 使用者路由
├── openapi.rs                       # OpenAPI/Swagger 文件生成
└── time.rs                          # 時間相關工具
```

| 檔案/目錄 | 職責 |
|------|------|
| `main.rs` | 啟動 HTTP Server、初始化所有依賴並綁定埠號（精簡版，啟動邏輯已移至 `startup/`） |
| `lib.rs` | 匯出模組，供整合測試與 bin target 使用 |
| `config.rs` | 從環境變數/`.env` 讀取並解析組態 (database URL、JWT secret 等) |
| `constants.rs` | 定義全域常數，如分頁預設值、狀態碼 |
| `error.rs` | 統一 `AppError` 型別，實作 `IntoResponse` 以回傳一致的錯誤 JSON |
| `routes/` | 集中管理所有 API 路由，依業務域拆分為 10 個子模組，`mod.rs` 組裝 Axum `Router` |
| `openapi.rs` | 自動產生 OpenAPI 3.0 文件供 Swagger UI 使用 |
| `time.rs` | 封裝時區轉換、時間格式化等工具函式 |

## 5. 架構層級職責對照表

下表總結 Backend 的分層架構與各層職責邊界：

| 層級 | 目錄 | 職責 | 不應包含 |
|------|------|------|----------|
| **Router** | `routes/` | 路由註冊、中間件掛載（依業務域拆分 10 子模組） | 業務邏輯、資料存取 |
| **Middleware** | `middleware/` | 橫切關注點（驗證、限流、快取） | 業務邏輯 |
| **Handler** | `handlers/` | 請求解析、參數驗證、回應組裝 | 資料庫操作、業務規則 |
| **Service** | `services/` | 業務邏輯、狀態機、跨模組協調、存取控制 | HTTP 相關概念、直接撰寫 SQL |
| **Repository** | `repositories/` | 封裝可複用 SQL 查詢（sqlx::query） | 業務邏輯判斷 |
| **Model** | `models/` | 資料結構、DTO、列舉定義 | 業務邏輯、資料存取 |
| **Startup** | `startup/` | 初始化、Migration、Seed、Server 啟動 | 請求處理邏輯 |
| **Utils** | `utils/` | 通用純函式工具（保留擴充空間） | 業務邏輯、AppState 依賴 |

> **設計原則**：依賴方向為 `Handler → Service → Repository → Model`，禁止反向依賴。中間件與工具層為跨層共用設施。
