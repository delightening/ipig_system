# 產品匯入：LLM 產生 SKU 準則

本文件提供給 LLM 使用的準則，讓其能根據產品名稱與規格正確推斷 `category_code` 與 `subcategory_code`，供匯入流程產生 SKU。

---

## 1. SKU 結構說明

- **格式**：`{category_code}-{subcategory_code}-{流水號}`
- **範例**：`CON-GAU-001`（耗材-紗布敷料-001）
- **流水號**：由系統依序產生，LLM 只需輸出 `category_code` 與 `subcategory_code`

## 3. 判斷優先順序與規則

1. **優先判斷主品類（category_code）**
   - 若名稱明顯屬於藥品（藥物名、療效）→ DRG
   - 若為注射/穿刺/縫合等侵入性醫材 → MED
   - 若為一次性消耗品（手套、紗布、棉棒、酒精）→ CON
   - 若為試劑、溶劑、標準品等 → CHM
   - 若為儀器、工具、零件 → EQP

2. **邊界案例**
   - **紗布**：用於傷口敷料 → MED-BND；用於一般清潔/實驗 → CON-GAU
   - **手套**：手術/檢診用 → MED-GLV；實驗/清潔用 → CON-GLV
   - **生理食鹽水**：若為注射用 → MED-OTH；若為清洗/稀釋用 → CHM-SOL 或 CON-OTH
   - **酒精**：消毒用 → CON-CLN；溶劑用 → CHM-SOL

3. **不確定時**
   - 使用 `OTH`（其他）子類
   - 主品類仍盡量依名稱判斷；若完全無法判斷，使用 `GEN`（通用）＋`OTH`

## 4. 輸出格式

LLM 應對每筆匯入列輸出：

```json
{
  "row": 2,
  "name": "紗布",
  "spec": "4x4",
  "category_code": "CON",
  "subcategory_code": "GAU",
  "confidence": 0.95,
  "reason": "紗布為耗材，子類紗布敷料"
}
```

- `row`：Excel/CSV 列號
- `name`、`spec`：與原始資料一致
- `category_code`、`subcategory_code`：依準則推斷
- `confidence`：0–1，信心度（可選）
- `reason`：簡短說明判斷依據（可選，供人工覆核）

---

## 5. 特殊注意事項

- 所有代碼為大寫 3 碼，例如：`DRG`、`ABX`、`OTH`
- 代碼必須與系統 `sku_categories`、`sku_subcategories` 表一致
- 若系統無對應子類，應選擇該主類下最接近的 `OTH`
