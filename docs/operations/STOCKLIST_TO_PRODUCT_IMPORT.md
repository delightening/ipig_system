# 耗材藥品庫存清表 → 產品匯入範本轉換說明

## 用途

將「耗材/藥品庫存表」CSV（如 `20260305_TU-04-00-01_耗材藥品庫存清表(年)_V.0315(TEST).csv`）轉成系統產品匯入範本格式，以便在「產品管理」中批次匯入品項。

## 使用方式

```bash
node scripts/stocklist-to-product-import.js [輸入檔.csv] [輸出檔.csv]
```

- 未帶參數時：讀取專案根目錄的 `20260305_TU-04-00-01_耗材藥品庫存清表(年)_V.0315(TEST).csv`，輸出 `product_import_from_stocklist.csv`。
- 範例：`node scripts/stocklist-to-product-import.js "你的庫存表.csv" "匯入用.csv"`

產出檔為 **UTF-8（含 BOM）**，可直接用 Excel 開啟後再上傳至系統「匯入產品」。

---

## 欄位對照

| 匯入範本欄位 | 來源（庫存表） | 說明 |
|-------------|----------------|------|
| **名稱** | 品名 + 廠商 | 有廠商時為「品名(廠商)」，區分不同廠商為不同產品 |
| **規格** | 規格 | 可為空 |
| **品類代碼** | — | **留空**，由系統預設 GEN |
| **子類代碼** | — | **留空**，由系統預設 OTH |
| **單位** | 單位 | 保留原文（包/瓶/盒/個/件/條/片/捲/袋/張/支等）；後端不驗證白名單，任意非空字串皆可 |
| **追蹤批號** | — | 固定填 `true` |
| **追蹤效期** | — | 固定填 `true` |
| **安全庫存** | 數量（安全庫存欄） | 對應庫存表「數量」欄 |
| **備註** | 原備註或廠商 | 有廠商時寫入「廠商: XXX」 |

## 疑義與建議

### 1. 品類/子類由系統判定

- 轉換時**不填**品類代碼、子類代碼，匯入時後端會使用預設 **GEN-OTH**，SKU 格式為 `GEN-OTH-001` 等。
- 品類僅影響 SKU 前綴與列表顯示名稱；若日後要改為依名稱/規格自動推斷品類，可於後端或匯入流程擴充邏輯。

### 2. 單位是否只吃代碼（已確認）

- **後端產品匯入僅檢查「單位」非空**，沒有單位代碼白名單；包、瓶、盒、個、件、條、片、捲、袋、張、支等皆可接受。
- 轉換腳本保留庫存表原文，無需改為 PCS/BX 等代碼。

### 3. 不同廠商分開

- 轉換腳本已將**有廠商**的品項名稱改為「品名(廠商)」，例如「醫用黏性膠帶(揚克)」「醫用黏性膠帶(全和)」會成為兩筆產品，不會被重複預檢合併。

### 4. 空白安全庫存

- 原表若「數量」為空，轉出之「安全庫存」為空；系統接受空值。
- 若希望一律有數字，可在 Excel 中將空白改為 0 或所需預設值。

### 5. 僅轉出「品項」、不包含庫存數量與效期

- 此轉換僅產出**產品主檔**匯入用欄位（名稱、規格、單位、安全庫存等）。
- 庫存數量、效期、批號、倉別等需在產品建立後，於「庫存/倉儲」或相關單據中另行登錄。
