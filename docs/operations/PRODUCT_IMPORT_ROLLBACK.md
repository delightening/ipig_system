# 產品匯入後還原

> **適用情境**：匯入產品時做錯（錯檔、錯選項、重複匯入等），需要回到匯入前的狀態。

---

## 重要：目前系統沒有「一鍵還原匯入」功能

產品匯入**不會**記錄「這次匯入建立了哪些產品 ID」，因此無法自動復原單次匯入。可採以下方式處理。

---

## 做法一：有匯入前備份（最乾淨）

若在匯入前有做資料庫備份，可直接還原到該時間點。

1. **確認備份檔**  
   例如：`backup_before_restore_YYYYMMDD_HHMMSS.dump` 或 `ipig_YYYYMMDD_HHMMSS.sql.gz`。

2. **還原**  
   - 使用專案還原腳本：見 `docs/database/RESTORE_OLD_DUMP.md`、`scripts/restore_old_dump.ps1`。  
   - 或依 `docs/scripts/backup/BACKUP.md` 的「還原」章節操作。

3. **注意**  
   還原會覆蓋**整個資料庫**，匯入之後到還原前的新增/修改都會消失，請確認可接受再執行。

**建議**：日後進行大量產品匯入前，先手動做一次備份（見下方「預防：匯入前備份」）。

---

## 做法二：沒有備份——手動還原（依建立時間）

若沒有匯入前備份，只能依「匯入大約時間」找出並停用該批產品（系統為軟刪除：`is_active = false`）。

### 步驟 1：確認匯入時間

- 從**稽核日誌**查「匯入產品」活動時間（例如：`ERP` / `PRODUCT_IMPORT`）。  
- 或依自己記得的大致時間（例如：今天 14:00 匯入的）。

### 步驟 2：在資料庫中停用該時段建立的產品

在 **psql** 或 **pgAdmin** 連到同一資料庫後執行（請改為你的實際時間）：

```sql
-- 範例：停用 2026-03-05 14:00 之後建立的產品（請改為你的匯入開始時間）
UPDATE products
SET is_active = false, updated_at = NOW()
WHERE created_at >= '2026-03-05 14:00:00+00';
```

- 若只匯入一批，可用單一時間點（如上）。  
- 若當天有多批匯入，可改成一段區間，例如：  
  `WHERE created_at >= '2026-03-05 14:00:00+00' AND created_at < '2026-03-05 15:00:00+00'`  
  再依實際時間調整。

### 步驟 3：確認結果

- 到 **產品管理** 列表查看，預設會只顯示啟用產品；被設為停用的產品不會出現在一般列表中，等於「還原」掉該批匯入的顯示與使用。  
- 若發現停用範圍不對（多停或少停），可再對該批產品執行 `UPDATE products SET is_active = true WHERE id IN (...)` 或依時間再跑一次 UPDATE 修正。

### 注意

- 系統的「刪除產品」是**軟刪除**（只設 `is_active = false`），不會刪除庫存、單據等關聯資料。  
- 若該批產品已有**採購單、入庫、庫存**等，停用產品後這些紀錄仍會存在，只是產品在列表中會被隱藏、無法再被選用。若需要一併還原庫存/單據，需另依業務需求處理（例如手動調整或由管理員從後台處理）。

---

## 做法三：沒有備份——在畫面上逐筆停用/刪除

若匯入筆數不多，且能從畫面上辨識出「這次匯入的」產品：

1. 到 **產品管理**，用篩選（品類、關鍵字等）縮小範圍。  
2. 逐筆進入產品詳情，使用**停用**或**刪除**（即軟刪除）。  
3. 系統沒有「批次勾選多筆再刪除」時，需逐筆操作。

---

## 預防：匯入前備份

大量匯入前建議先做一次手動備份，之後若要還原，用「做法一」即可。

```powershell
# 若使用 Docker，例如（請依實際容器名稱與 DB 名稱調整）：
docker compose exec db pg_dump -U postgres ipig_db -Fc -f /tmp/pre_import.dump
# 再從容器複製到主機保留
docker cp ipig-db:/tmp/pre_import.dump ./backup_pre_product_import_YYYYMMDD.dump
```

或依 `docs/scripts/backup/BACKUP.md` 的「手動備份」方式執行。

---

## 小結

| 情境 | 建議做法 |
|------|----------|
| 有匯入前備份 | 還原該備份（做法一） |
| 沒有備份、知道匯入大約時間 | 用 SQL 依 `created_at` 將該批產品設為停用（做法二） |
| 沒有備份、筆數少且能辨識 | 在產品管理逐筆停用/刪除（做法三） |

日後若系統新增「匯入批次」記錄（例如記錄每次匯入的產品 ID），即可實作一鍵還原該批匯入。
