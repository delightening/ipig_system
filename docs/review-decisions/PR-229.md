## Breaking Changes 評估

react-router v7 是 **重大整合**（v6 + Remix data router 統一）：

### 主要 breaking

1. **Module 結構**：v7 統一為 `react-router`（不是 `react-router-dom`），但 6.x 有 deprecation alias，v7 可能仍 export 但 type 來源變
2. **Type names 重整**：
   - `LoaderFunction` / `ActionFunction` 簽名擴展
   - `RouteObject` 型別細節改動
   - `useNavigate` / `useLocation` 等 hook 型別繼承變動
3. **Future flags 強制**：v6 的 `future: { v7_*: true }` flags 在 v7 為 default 行為
4. **NavLink children render prop API** 微調
5. **Framework mode** 引入（loader/action SSR），純 client app 雖可選擇不用，但 type 層仍受影響

### 影響面 vs 適配工作量

71 檔案 import — 多為 `useNavigate` / `useLocation` / `useParams` / `Link` / `Navigate` / `Outlet` 標準 hook + component。實際 **runtime 行為大致兼容**（CRA-style client routing 仍 work），但 **type 層需要逐一檢視**：

- 估計 1-2 個 TypeScript type 修補 patch（types/navigation.ts + 部分頁面）
- 1-2 處 future flag config 移除（main.tsx / App.tsx 的 RouterProvider config）
- **預估工作量：4-8 小時**（含 typecheck + lint + 跑一輪 dev server smoke 主要 routes）
