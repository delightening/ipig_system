# 手機端 Dialog 滾動與縮放修復計劃

## 問題描述

在手機端使用系統時發現兩個問題：
1. **Dialog 無法上下滾動**：長表單 Dialog 內容超出螢幕高度時無法捲動
2. **螢幕縮放比例偏差**：行動裝置上可能出現意外縮放行為

## 根因分析

### Dialog 滾動問題
- 基礎 [dialog.tsx](file:///d:/Coding/ipig_system/frontend/src/components/ui/dialog.tsx) 的 `DialogContent` 使用 `fixed + translate(-50%, -50%)` 置中定位
- **未設定** `max-height` 與 `overflow-y: auto`，導致內容超出視窗時無法捲動
- 部分 Dialog 已個別補上（如 `SurgeryFormDialog` 有 `max-h-[90vh] overflow-y-auto`），但多數 Dialog 未處理
- 約有 **20+ 處** `DialogContent` 缺少滾動處理

### 縮放問題
- [index.html](file:///d:/Coding/ipig_system/frontend/index.html) viewport 僅設 `width=device-width, initial-scale=1.0`
- 缺少 `maximum-scale=1, user-scalable=no`，某些手機瀏覽器在表單 input focus 時會自動放大
- Dialog 內的 `<input>` 字體小於 16px 時，iOS Safari 會自動縮放頁面

## 提案變更

### 1. 修改基礎 DialogContent 元件

#### [MODIFY] [dialog.tsx](file:///d:/Coding/ipig_system/frontend/src/components/ui/dialog.tsx)

**變更內容：**
- 將 `DialogContent` 的定位從 `fixed + translate` 改為 `fixed inset-0` + flexbox 置中
- 內建 `max-h-[90vh]` / 手機端 `max-h-[100dvh]` + `overflow-y-auto`
- 分離 Header / Body / Footer 區域，確保只有內容區域可滾動

```diff
- "fixed left-[50%] top-[50%] z-50 grid w-full max-w-lg translate-x-[-50%] translate-y-[-50%] ..."
+ "fixed inset-0 z-50 flex items-center justify-center p-4 sm:p-6"
```

DialogContent 內部結構改為：
```tsx
<div className="relative w-full max-w-lg max-h-[100dvh] sm:max-h-[90vh] overflow-y-auto bg-background border shadow-lg sm:rounded-lg p-6">
  {children}
  <Close button />
</div>
```

> [!IMPORTANT]
> 這是全域元件修改，會影響所有使用 `DialogContent` 的頁面。由於改為 flexbox 置中 + 內部 div 滾動，視覺外觀基本不變，但需要確認所有 Dialog 仍正常運作。

### 2. 修改 viewport 設定

#### [MODIFY] [index.html](file:///d:/Coding/ipig_system/frontend/index.html)

```diff
- <meta name="viewport" content="width=device-width, initial-scale=1.0" />
+ <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />
```

### 3. 加入手機端 Dialog 全域 CSS

#### [MODIFY] [index.css](file:///d:/Coding/ipig_system/frontend/src/index.css)

新增手機端 Dialog 專屬樣式：
- 在小螢幕上，Dialog 佔據更多螢幕空間（減少 padding）
- 輸入欄位字體不小於 16px（防止 iOS 自動縮放）
- Touch 裝置 `-webkit-overflow-scrolling: touch` 流暢捲動
- Body overlay 時禁止背景滾動

### 4. 清理各 Dialog 的重複樣式

由於基礎元件已內建滾動支援，移除各 Dialog 重複的 `max-h-[90vh] overflow-y-auto` class：
- `SurgeryFormDialog.tsx`
- `SacrificeFormDialog.tsx`
- `ObservationFormDialog.tsx`
- `QuickEditAnimalDialog.tsx`
- `BloodTestTab.tsx`（2 處）
- `ProtocolEditPage.tsx`
- 其他已有 `max-h-[...]` 的 Dialog

> [!NOTE]
> 若個別 Dialog 需要更大的寬度（如 `max-w-4xl`），這些 class 保留不動。僅移除與基礎元件重複的高度/滾動 class。

## 驗證計劃

### 手動驗證（需使用者協助）
1. 在手機瀏覽器開啟系統
2. 打開各種 Dialog（手術紀錄、觀察紀錄、犧牲表單等長表單）
3. 確認可以上下滾動
4. 確認輸入欄位 focus 時不會意外縮放
5. 確認 Dialog 關閉按鈕在捲動後仍可見/可點擊

### 桌面端迴歸驗證
1. 確認桌面瀏覽器上所有 Dialog 外觀不變
2. 確認長 Dialog 仍可滾動
3. 使用 Chrome DevTools 手機模擬器檢查各斷點

> [!CAUTION]
> 由於是全域元件修改，建議先部署至測試環境，讓使用者在手機端實際操作確認。
